# 在线一对一私密视频聊天平台 - 技术方案

## 项目概述

**项目名称**: o2o (One-to-One)
**定位**: 一对一私密在线视频聊天平台（纯浏览器方案）
**技术栈**: Next.js 14+ + Clerk + Cloudflare 全家桶

---

## 一、技术架构

### 1.1 核心技术选型

| 层级 | 技术 | 说明 |
|-----|------|------|
| 框架 | Next.js 15 (App Router) | 支持 SSR/SSG，文件路由系统，React 19 |
| 语言 | TypeScript | 类型安全，全栈统一 |
| 样式 | TailwindCSS + shadcn/ui | 原子化 CSS + 高质量组件库 |
| **认证** | **Clerk** | 托管式用户认证，支持多因子 |
| 视频 SDK | Daily.co | WebRTC 托管服务，内置录制/虚拟背景 |
| 邮件服务 | Resend | 邮件发送，用于通知/转化 |
| 后端 | Next.js API Routes | Serverless Function |
| 数据库 | Cloudflare D1 + Drizzle ORM | SQLite 边缘数据库 |
| 文件存储 | Cloudflare R2 | S3 兼容对象存储 |
| **部署** | **Cloudflare Pages** | 全球 CDN，自动 HTTPS，Next.js 官方支持 |
| CI/CD | Cloudflare Pages CI | Git 推送自动构建部署 |

## 二、为什么选择 Clerk

### 2.1 Clerk vs NextAuth/Auth.js

| 特性 | Clerk | NextAuth/Auth.js |
|-----|-------|-----------------|
| 部署方式 | 托管服务 | 自部署 |
| 配置复杂度 | 低（SDK 集成） | 中（需配置 Provider） |
| 多因子认证 | ✅ 内置 | ⚠️ 需自行实现 |
| 用户管理后台 | ✅ 内置 | ❌ 无 |
| 会话管理 | ✅ 托管 | ⚠️ 需处理 JWT/DB |
| 社交登录 | ✅ 一键配置 | ⚠️ 需分别配置 OAuth |
| 边缘兼容 | ✅ 完美支持 | ⚠️ 部分 Provider 不兼容 |
| Cloudflare Pages | ✅ 官方支持 | ⚠️ 需额外配置 |
| 成本 | $25/月起 | 免费 |

### 2.2 Clerk 核心优势

1. **零运维**: 无需担心认证服务器、密钥轮换、安全补丁
2. **快速集成**: 5 行代码完成登录功能
3. **开箱即用**: 登录/注册/忘记密码/多因子全都有
4. **边缘友好**: 与 Cloudflare Workers/Pages 深度集成
5. **多租户支持**: 内置组织/角色管理（未来扩展用）

### 2.3 Clerk 定价

| 套餐 | 价格 | 包含内容 |
|-----|------|---------|
| Free | $0 | 10,000 MAU + 基础功能 |
| Pro | $25/月 | 无限 MAU + 多因子 + 组织管理 |
| Enterprise | 定制 | SSO + SLA + 专属支持 |

**MVP 阶段**: Free 套餐足够（1 万月活用户）
**成长期**: Pro 套餐 $25/月

## 三、为什么选择 Cloudflare 部署

### 3.1 Cloudflare Pages vs Vercel

| 特性 | Cloudflare Pages | Vercel |
|-----|-----------------|--------|
| 带宽免费额度 | 无限 | 100GB/月 |
| 构建时长限制 | 30 分钟 | 5 分钟 |
| Serverless 超时 | 15 分钟 | 10-60 秒 |
| 全球边缘节点 | 300+ | 100+ |
| D1 数据库集成 | ✅ 原生 | ⚠️ 需配置 |
| R2 存储集成 | ✅ 原生 | ⚠️ 需配置 |
| 免费额度 | 更高 | 较友好 |
| 中国区访问 | 较好 | 一般 |

### 3.2 Cloudflare 全家桶优势

1. **一站式解决**: 部署 + 数据库 + 存储 + CDN 全部搞定
2. **成本可控**: 免费额度充足，MVP 阶段几乎零成本
3. **性能优秀**: 边缘计算，全球 300+ 节点
4. **原生集成**: D1/R2/Pages 无缝协作
5. **安全内置**: DDoS 防护、WAF、速率限制

### 3.3 Cloudflare 集成方案

## 四、数据库设计

### 4.1 D1 数据库 Schema (Drizzle ORM)

```typescript
// drizzle/schema.ts
import { sqliteTable, text, integer } from 'drizzle-orm/sqlite-core';

// 用户表 (Clerk 用户数据同步)
export const users = sqliteTable('users', {
  id: text('id').primaryKey(),        // Clerk user ID
  email: text('email').notNull(),
  username: text('username'),
  avatarUrl: text('avatar_url'),
  createdAt: integer('created_at', { mode: 'timestamp' }).notNull(),
  updatedAt: integer('updated_at', { mode: 'timestamp' }).notNull(),
});

// 房间表
export const rooms = sqliteTable('rooms', {
  id: text('id').primaryKey(),
  name: text('name').notNull(),
  hostId: text('host_id').notNull(),  // 房主 ID
  status: text('status').notNull(),   // waiting, active, ended
  maxParticipants: integer('max_participants').default(2),
  startedAt: integer('started_at', { mode: 'timestamp' }),
  endedAt: integer('ended_at', { mode: 'timestamp' }),
  createdAt: integer('created_at', { mode: 'timestamp' }).notNull(),
});

// 房间参与者表
export const participants = sqliteTable('participants', {
  id: text('id').primaryKey(),
  roomId: text('room_id').notNull().references(() => rooms.id),
  userId: text('user_id').notNull().references(() => users.id),
  role: text('role').notNull(),       // host, guest
  joinedAt: integer('joined_at', { mode: 'timestamp' }).notNull(),
  leftAt: integer('left_at', { mode: 'timestamp' }),
});

// 订阅计划表（会员升级）
export const subscriptions = sqliteTable('subscriptions', {
  id: text('id').primaryKey(),
  userId: text('user_id').notNull().references(() => users.id),
  plan: text('plan').notNull(),       // free, member
  status: text('status').notNull(),   // active, cancelled, expired
  startDate: integer('start_date', { mode: 'timestamp' }).notNull(),
  endDate: integer('end_date', { mode: 'timestamp' }),
});

// 用户时长账户表
export const userCredits = sqliteTable('user_credits', {
  id: text('id').primaryKey(),
  userId: text('user_id').notNull().references(() => users.id),
  totalMinutes: integer('total_minutes').notNull(),      // 总分钟数
  usedMinutes: integer('used_minutes').notNull().default(0),  // 已用分钟数
  isLifetime: integer('is_lifetime', { mode: 'boolean' }).notNull().default(true), // 是否永久有效
  createdAt: integer('created_at', { mode: 'timestamp' }).notNull(),
});

// 订单表
export const orders = sqliteTable('orders', {
  id: text('id').primaryKey(),
  userId: text('user_id').notNull().references(() => users.id),
  stripePaymentIntentId: text('stripe_payment_intent_id').unique(),
  package: text('package').notNull(),         // starter, standard, unlimited
  minutes: integer('minutes').notNull(),
  amount: integer('amount').notNull(),        // 支付金额（美分）
  currency: text('currency').notNull(),       // usd
  status: text('status').notNull(),           // pending, succeeded, failed, refunded
  createdAt: integer('created_at', { mode: 'timestamp' }).notNull(),
});

// 免费试用表
export const freeTrials = sqliteTable('free_trials', {
  id: text('id').primaryKey(),
  userId: text('user_id').notNull().references(() => users.id),
  remainingMinutes: integer('remaining_minutes').notNull(),  // 剩余分钟数
  expiresAt: integer('expires_at', { mode: 'timestamp' }).notNull(),
  usedMinutes: integer('used_minutes').notNull().default(0),
});

// 会议邀请令牌表 (一次性)
export const inviteTokens = sqliteTable('invite_tokens', {
  id: text('id').primaryKey(),
  roomId: text('room_id').notNull().references(() => rooms.id),
  token: text('token').notNull().unique(),  // 一次性令牌
  used: integer('used', { mode: 'boolean' }).notNull().default(false),
  joinedBy: text('joined_by'),              // 加入者邮箱 (可选)
  createdAt: integer('created_at', { mode: 'timestamp' }).notNull(),
  expiresAt: integer('expires_at', { mode: 'timestamp' }).notNull(),
});

// 会议记录表
export const meetingRecords = sqliteTable('meeting_records', {
  id: text('id').primaryKey(),
  roomId: text('room_id').notNull().references(() => rooms.id),
  hostId: text('host_id').notNull(),
  guestId: text('guest_id'),                // 可为空 (免登录用户)
  duration: integer('duration').notNull(),  // 时长 (秒)
  type: text('type').notNull(),             // audio, video
  status: text('status').notNull(),         // active, ended, abandoned
  endedBy: text('ended_by'),                // host, guest, system
  createdAt: integer('created_at', { mode: 'timestamp' }).notNull(),
});

// 邮件日志表 (Resend)
export const emailLogs = sqliteTable('email_logs', {
  id: text('id').primaryKey(),
  userId: text('user_id').notNull().references(() => users.id),
  type: text('type').notNull(),             // welcome, trial_expiring, upgrade
  status: text('status').notNull(),         // sent, delivered, failed
  sentAt: integer('sent_at', { mode: 'timestamp' }),
  metadata: text('metadata'),               // JSON 额外信息
});
```

```typescript
import DailyIframe from '@daily-co/daily-js';

export async function createDailyRoom(roomName: string) {
  // 调用后端 API 创建房间
  const res = await fetch('/api/daily-room', {
    method: 'POST',
    body: JSON.stringify({ roomName }),
  });
  return res.json();

```

```typescript
// components/video-room.tsx
'use client';

import { useEffect, useRef } from 'react';
import DailyIframe from '@daily-co/daily-js';

export function VideoRoom({ roomUrl }) {
  const iframeRef = useRef<HTMLIFrameElement>(null);

  useEffect(() => {
    const callFrame = DailyIframe.createFrame(iframeRef.current);
    callFrame.join({ url: roomUrl });
    return () => callFrame.destroy();
  }, [roomUrl]);

  return <iframe ref={iframeRef} style={{ width: '100%', height: '100%' }} />;
}
```

### 7.2 Cloudflare Pages 部署

**前置条件**：
- Cloudflare 账号
- Git 仓库（GitHub/GitLab）
- 已创建 D1 数据库和 R2 存储桶

**部署步骤**：

```bash
# 1. 登录 Cloudflare
npx wrangler login

# 2. 创建 Pages 项目
npx wrangler pages project create o2o \
  --production-branch=main \
  --build-command="npm run build" \
  --output-directory=".vercel/output/static"

# 3. 绑定 D1 数据库
npx wrangler pages project update o2o \
  --d1-db="o2o-db"

# 4. 绑定 R2 存储桶
npx wrangler pages project update o2o \
  --r2-bucket="o2o-storage"

# 5. 配置环境变量（敏感信息）
npx wrangler pages secret put CLERK_SECRET_KEY
npx wrangler pages secret put DAILY_API_KEY
npx wrangler pages secret put RESEND_API_KEY
npx wrangler pages secret put STRIPE_SECRET_KEY
```

**wrangler.toml 配置**：
```toml
name = "o2o"
compatibility_date = "2024-01-01"
pages_build_output_dir = ".vercel/output/static"

[[d1_databases]]

database_id = "your-d1-database-id"

[[r2_buckets]]
binding = "BUCKET"
bucket_name = "o2o-storage"
```

**next.config.js 配置**：

### 10.5 风控策略

**IP 绑定机制**：
```
1. 创建会议时生成一次性 token
2. guest 首次点击链接时，记录 IP 地址
3. token 与该 IP 绑定
4. 后续请求验证 IP 是否匹配
5. 会议结束后 token 失效
```

**限制**：
- 同一 IP 24 小时内最多创建 3 个会议（防滥用）
- 同一邮箱/设备 7 天内只能试用一次

### 10.4 会议规则
- **两人限制**: 只允许 1 对 1，第三人无法加入
- **一次性令牌**: 分享链接包含一次性 token，使用后失效
- **IP 绑定**: token 首次使用时绑定 IP，防止链接被转发
- **免登录加入**: 第三方通过链接可直接参会，无需注册
- **默认语音**: 进入会议时摄像头默认关闭
- **会议结束**: 任意一方离开，会议立即结束，房间不可复用

### 10.11 时长账户逻辑

**扣费规则**：
```

3. 会议结束（任意一方离开）停止扣费

```

**余额不足处理**：
```
1. 剩余<5 分钟时弹出提醒
2. 余额为 0 时自动结束会议
3. 引导充值后继续
```

**有效期管理**：
```
- 免费赠送：7 天有效
- 标准包/超值包/无限包：永久有效
- 余额不足时 (<5 分钟) 弹窗提醒
- 余额为 0 时自动结束会议
```
